## Troubleshooting Hub

Top-level entry point for self-service Twingate troubleshooting. Pointer doc to the broader help ecosystem.

### Resources

| Resource | Purpose |
|---|---|
| **Twingate Help Center** | Comprehensive Knowledge Base with detailed troubleshooting articles maintained by the Technical Support Engineering team |
| **Self-Service Troubleshooting Guide** | Best starting point for common issues -- in Help Center |
| **Knowledge Base articles for log collection** | How to gather Client logs and Connector logs (see /docs/managing-devices, Connector logging docs) |
| **Twingate Subreddit** (r/Twingate) | Community support; Twingate staff and expert users respond |
| **Free enablement content** | Self-paced learning materials |

### Where to Start by Failure Type

- **User-side / device issues**: /docs/device-failures

- **P2P specifically**: /docs/troubleshooting-p2p

### When to Use This Decision Tree

- /docs/how-to-troubleshoot -- Methodology + symptom-to-cause table

### Decision Notes

- Self-service first: most issues have known patterns documented in the KB / per-failure-mode pages
- Community support is free; technical support is plan-gated (Teams/Business/Enterprise)
- For tickets to Support: gather Client + Connector logs first (massively reduces back-and-forth)
