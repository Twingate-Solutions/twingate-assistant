# Getting Started with DigitalOcean and Twingate

## Summary
Deploys a Twingate Connector on a DigitalOcean Droplet using `doctl` CLI and `cloud-init`. Enables secure remote access to private DigitalOcean resources. Setup is completed via the Twingate Admin Console token generation flow combined with a single terminal command.

## Prerequisites
- DigitalOcean account with API access
- `doctl` CLI installed and authenticated with DigitalOcean credentials
- Twingate account with Admin Console access
- Basic familiarity with DigitalOcean Droplets and `cloud-init`

## Step-by-Step

1. **Generate Connector tokens** in Twingate Admin Console:
   - Navigate to **Remote Networks** → select target network
   - Add or select an undeployed Connector → **See More**
   - Select **DigitalOcean** option
   - Scroll to **Step 2** → click **Generate Tokens** (authenticate when prompted)
   - Scroll to **Step 4** → copy the installation command

2. **Deploy via doctl**: Paste the copied command into terminal; it creates a Droplet with Connector installed via `cloud-init`

3. **Verify in DigitalOcean**: Check Droplet is running in Control Panel → Droplets section

4. **Verify in Twingate Admin Console**: Remote Networks → select network → select Connector → confirm **Controller** and **Relay** statuses show `connected`

## Configuration Values
- Tokens are auto-generated by Admin Console and embedded in the `doctl` command
- Each Connector requires its own unique Access Token + Refresh Token pair (never reuse)

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Token errors | Verify tokens are correctly entered in `cloud-init` script |
| Copy/paste errors | Paste command into a separate script file first (whitespace issues) |
| Connectivity problems | Run `doctl compute droplet list`; check Connector service status |
| `doctl` not found | Install/authenticate `doctl` per DigitalOcean docs |

## Gotchas
- **Never reuse token sets** — each Connector must have its own unique token pair
- Terminal whitespace interpretation can corrupt the pasted `doctl` command; use an intermediate script file if needed
- Connector won't show `connected` until Droplet has fully initialized via `cloud-init`
