# Getting Started with DigitalOcean and Twingate

## Summary
Deploy a Twingate Connector on a DigitalOcean Droplet using `doctl` CLI and `cloud-init`. The process generates connector tokens from the Twingate Admin Console and runs a single command to create and configure the Droplet automatically.

## Key Information
- Deployment uses a `cloud-init` script embedded in a `doctl` command generated by the Admin Console
- Each Connector requires its own unique Access/Refresh token pair (no reuse)
- Verification requires checking both DigitalOcean Droplet status and Twingate Admin Console connector status

## Prerequisites
- DigitalOcean account with API access
- `doctl` CLI installed and authenticated with DigitalOcean credentials
- Twingate account with Admin Console access
- Existing Remote Network configured in Twingate

## Step-by-Step

1. **Generate Connector Tokens**
   - Admin Console → Remote Networks → select network
   - Add or select an undeployed Connector → See More → DigitalOcean
   - Step 2: click **Generate Tokens** (authenticate when prompted)
   - Step 4: copy the generated installation command

2. **Deploy via doctl**
   - Paste the copied command into terminal
   - Command creates a Droplet with Connector pre-configured via `cloud-init`

3. **Verify Deployment**
   - DigitalOcean: Control Panel → Droplets → confirm Droplet is running
   - Twingate: Admin Console → Remote Networks → Connector → confirm **Controller** and **Relay** statuses show `connected`

## Configuration Values
- Tokens are embedded directly in the generated `doctl` command (not set manually)
- No manual environment variables required; `cloud-init` handles configuration

## Gotchas
- **Never reuse token sets** — each Connector must have unique Access/Refresh tokens
- Terminal whitespace handling may corrupt the pasted command; copy to a script file first if issues occur
- Verify Droplet creation with: `doctl compute droplet list`

## Troubleshooting
| Issue | Fix |
|-------|-----|
| Token errors | Verify tokens are correctly included in `cloud-init` script |
| `doctl` not found | Install/authenticate `doctl` per DigitalOcean docs |
| Connectivity problems | Check Droplet running status + Connector service on Droplet |
