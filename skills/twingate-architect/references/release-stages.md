## Twingate Release Stages

Twingate releases new features in two stages: Beta (available for testing, limited support, may have bugs and breaking changes) and General Availability / GA (production-ready, full support, breaking changes avoided). This page defines what each stage means for reliability, support, documentation, and deployment eligibility.

**Key Information**

| Area | Beta | GA |
|---|---|---|
| Functionality | May be incomplete | Feature-complete, polished UI/UX |
| Reliability | Possible minor issues | Thoroughly tested |
| Stability | May have breaking changes | Breaking changes avoided |
| Performance | Not fully optimized | Optimized for production |
| Support | Not guaranteed | Full formal support |
| Documentation | May be incomplete | Comprehensive |
| Testing | Limited environments | Real-world environments |
| Availability | Opt-in or new networks only | New and existing networks |

**Gotchas**
- Beta features may have breaking changes between releases -- avoid relying on Beta in production automation without monitoring for changes
- Beta features may be opt-in only and not visible to all accounts by default
