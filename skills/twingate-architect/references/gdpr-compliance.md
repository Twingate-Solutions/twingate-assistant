# GDPR Compliance

## Page Title
GDPR Compliance

## Summary
Twingate complies with GDPR as both a data controller (for its own processing activities) and a data processor (on behalf of customers). Customers requiring GDPR compliance can execute a data processing addendum (DPA) with Twingate. Cross-border data transfers from EU/UK to the U.S. are covered via Standard Contractual Clauses (SCCs) and the UK International Data Transfer Addendum.

## Key Information
- Twingate operates under **dual roles**: data controller (own data) and data processor (customer data)
- Twingate collects **limited personal data** from customers
- A **Data Processing Addendum (DPA)** is available for customers needing GDPR compliance
- EU→US transfers rely on **Standard Contractual Clauses (SCCs)**
- UK→US transfers rely on the **UK International Data Transfer Addendum**
- Twingate maintains an ongoing GDPR compliance program

## Controller Obligations Twingate Fulfills
- Privacy notices where required
- Data subject rights request procedures
- Security measures for personal data
- Third-party processor due diligence and contractual terms

## Prerequisites
- Customer Agreement must be in place
- For GDPR-specific obligations: execute the **Data Processing Addendum** with Twingate

## Configuration / Documents
| Document | Purpose |
|---|---|
| Privacy Policy | Describes Twingate's controller-side data processing |
| Customer Agreement | Covers handling of customer and personal data |
| Data Processing Addendum | Required for GDPR-compliant customer relationships |
| UK International Data Transfer Addendum | UK GDPR cross-border transfer compliance |

## Gotchas
- Twingate is a **U.S.-based company** — EU/UK customers must ensure the DPA and SCCs are executed for lawful data transfers
- GDPR compliance is described as a **continuous process** — confirm current DPA/SCC versions are in use
- For detailed data types and processing activities, see the separate **Twingate & Customer Data** page (not covered here)
