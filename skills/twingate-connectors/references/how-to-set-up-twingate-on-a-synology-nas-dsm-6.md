# Deploy Twingate Connector on Synology NAS (DSM 6.x)

## Summary
Deploys a Twingate Connector on Synology NAS devices running DSM 6.x/7.1 or earlier using DSM's native Task Scheduler. Enables secure remote access to the NAS and other local network devices without VPN server setup or port forwarding.

## Key Information
- DSM is Linux-based; Connector runs via Docker deploy command
- Uses Task Scheduler (not Docker UI directly) to deploy and auto-upgrade
- Connector runs as `root` user in the scheduled task
- Verify success via Connector status in Admin Console (should show "Connected")
- After setup, add NAS as a Resource using its local IP address

## Prerequisites
- Synology NAS running DSM 6.x or 7.1 or earlier
- Remote Network created in Twingate Admin Console
- Connector created under that Remote Network (use pre-created ones)
- Docker deploy tokens generated in Admin Console
- Browser access to DSM web interface (`https://X.X.X.X:5001`)
- Must be on the same local network as the NAS during setup

## Step-by-Step

1. In Admin Console → Networks, open Remote Network details page
2. Select a pre-created Connector → click **Deploy Connector**
3. Click **Docker** → **Generate Tokens** → Authenticate
4. Configure optional toggles (custom DNS, local connection logging)
5. Copy the generated deploy command from the bottom of the page
6. Log into DSM web admin (`https://<NAS-IP>:5001`)
7. Open **Task Scheduler**
8. Create new task: **Scheduled Task → User-defined script**
9. **General tab**: Set task name (e.g., `Twingate Connector`), set user to `root`
10. **Schedule tab**: Adjust run frequency to minimize disruption (default: daily at midnight)
11. **Task Settings tab**: Paste the deploy command into **User-defined script**
12. Save → Select task → Click **Run** → Confirm prompt
13. Verify Connector shows "Connected" in Admin Console

## Configuration Values
- **Deploy method**: Docker (command generated by Admin Console)
- **DSM Task user**: `root` (required)
- **Default schedule**: Daily at midnight (adjust as needed)
- **DSM admin URL**: `https://X.X.X.X:5001`

## Gotchas
- The scheduled task handles both initial deployment and future upgrades — schedule timing affects upgrade disruption windows
- NAS must be added separately as a Resource using its **local IP address** after Connector is running
- This guide applies to DSM 6.x and 7.1 or earlier only — newer DSM versions have a separate guide
- Must confirm the run prompt when manually triggering the task
