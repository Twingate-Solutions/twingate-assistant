# Reviewing Access Requests

## Summary
Admins and designated Resource Approvers can approve or deny Access Requests generated by JIT access workflows or Usage-Based Auto Lock policies. The review process is identical regardless of which workflow triggered the request.

## Key Information
- Access Requests page shows all open requests and resolved requests from the last 90 days
- Red dot on bell icon in Admin Console indicates pending requests
- Resource Approvers only see the Access Requests page—no other Admin Console access
- Email notifications to Resource Approvers always include a direct link to the pending request

## Who Can Review Requests

**Admin roles (all Users & Resources):**
- Admin
- DevOps
- Access Reviewer

**Resource Approvers:**
- Members of Groups assigned to a specific Resource
- Can only approve requests for Resources they are assigned to
- No Admin Console access beyond Access Requests page
- Cannot configure notification preferences (always receive email)

## Prerequisites
- Admin Console access to assign Resource Approvers
- Groups must exist before assigning as Resource Approvers
- Users must be members of an assigned Approver Group to review requests

## Step-by-Step: Assigning Resource Approvers
1. Navigate to the Resource's detail page in Admin Console
2. Open the Resource Approvers assignment dialog
3. Add one or more Groups as Resource Approvers
4. Members of assigned Groups can now review requests for that Resource

## Step-by-Step: Reviewing Access Requests
1. Look for red dot on bell icon (upper right of Admin Console)
2. Select open request to navigate to Access Requests page
3. Alternatively, review from User detail page or Resource detail page
4. Approve or deny the request

**Resource Approver flow:** Click link in email notification → log in at tenant URL (e.g., `https://autoco.twingate.com`) → lands directly on Access Requests page

## Gotchas
- Resource Approvers **cannot** configure notifications—they always receive emails; plan Group assignments accordingly
- Resolved requests are only visible for **90 days**
- Resource Approvers logging into the standard Admin Console URL will only see the Access Requests page—this is expected behavior, not a bug
- Recommend creating Groups specifically tailored for delegation use cases rather than repurposing existing Groups
